## Naming Conventions

- Agent folders are lowercase and match the agent class name minus "Agent"
  (e.g. `ArchitectAgent` -> `architect/`, `CoderAgent` -> `coder/`)
- Domain files are lowercase and match the domain string used in the context dict
  (e.g. `context["domain"] == "timeseries"` -> `architect/timeseries.md`)
- `META_SKILL.md` is uppercase to visually distinguish it from domain skills
- Skills generated by the Skills Writer agent (phase 2) follow the same naming rules
  and are appended to the relevant agent folder

## Skill File Format

Every skill file must open with YAML frontmatter followed by a markdown body.
See META_SKILL.md for the full specification and a template.

Required frontmatter fields:
- `name`        вЂ” kebab-case identifier, unique within its agent folder
- `description` вЂ” one sentence: what this skill does and when the agent should apply it
- `agent`       вЂ” which agent loads this skill (e.g. `architect`, `coder`)
- `version`     вЂ” semver string, start at `1.0`
- `domains`     вЂ” list of applicable domains, or `[all]` for domain-agnostic skills

## Loading Rules (enforced by BaseAgent)

1. At agent init, the agent resolves its skill folder: `skills/<agent_name>/`
2. For domain-specific agents, the skill file is `skills/<agent_name>/<domain>.md`
3. For domain-agnostic agents (validator, critic), the skill file is named explicitly
4. The skill body is injected into the agent's system prompt at call time
5. If no skill file exists for the current domain, the agent runs without one (no crash)

## Writing New Skills

Before writing any skill, read META_SKILL.md in full.
Skills generated by the Skills Writer agent must also conform to META_SKILL.md.
